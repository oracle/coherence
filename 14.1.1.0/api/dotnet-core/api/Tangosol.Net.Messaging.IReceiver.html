Interface IReceiver

An IReceiver processes unsolicited IMessage objects sent via any number of IChannel objects.

Namespace: Tangosol.Net.Messaging
Assembly: Coherence.Core.dll
Syntax
public interface IReceiver
Remarks

An IReceiver acts as a server-side proxy, in that it can be registered with an IConnectionAcceptor, it can be looked up, and IChannels from multiple clients can be established to it. In this sense, the IReceiver represents server-side state shared across any number of client IChannels, and thus provides an efficient mechanism for demultiplexing multi-client communication into a shared service proxy, and locating state that is shared across all of those client IChannels. Conversely, the IChannel object represents client-specific state, allowing per-client information to be efficiently managed on the server side.

While the IReceiver is particularly useful as a server-side proxy, it is also useful on the client, allowing a client to publish named services to a server, and in the case of both named services and any other IChannels created by a client, it allows a client to efficiently manage stateful communication and process unsolicited IMessage objects.

RegisterChannel(IChannel)

Notify this IReceiver that it has been associated with a IChannel.

Declaration
void RegisterChannel(IChannel channel)
Parameters
Type Name Description
IChannel channel

An IChannel that has been associated with this IReceiver.
Remarks

This method is invoked by the IChannel when an IReceiver is associated with the IChannel.

Once registered, the IReceiver will receive all unsolicited IMessage objects sent through the IChannel until the IChannel is unregistered or closed. Without a IReceiver, the unsolicited IMessage objects are executed with only an IChannel as context; with an IReceiver, the IReceiver is given the IMessage to process, and may execute the IMessage in turn.
